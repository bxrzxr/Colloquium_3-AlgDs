# 1. Обход в глубину DFS
один из основных методов обхода графа часто используемый для проверки связности, поиска цикла и компонент сильной связности и для топологической сортировки.
## Идея:
Общая идея алгоритма состоит в следующем: для каждой не пройденной вершины необходимо найти все не пройденные смежные вершины и повторить поиск для них.

## Пошаговое представление:
1. Выбираем любую вершину из еще _не пройденных_, обозначим ее как u.
2. Запускаем процедуру dfs(u)
    - Помечаем вершину u как _пройденную_
    - Для каждой _не пройденной_ смежной с u вершиной (назовем ее v) запускаем dfs(v)
3. Повторяем шаги 1 и 2, пока все вершины не окажутся _пройденными_.
## Код
```cpp
void dfs(vector<vector<int>> &graphs, int v, vector<int> &visited){

	visited[v] = 1;
  
	for (int to : graphs[v]){
		if (!visited[to]){
			dfs(graphs, to, visited);
		}
	}
}
```

## DFS с покраской в цвета
Зачастую, простой информации "были/не были в вершине" не хватает для конкретных целей.
Поэтому в процессе алгоритма вершинам задают некоторые цвета:
- если вершина _белая_, значит, мы в ней еще не были, вершина _не пройдена_;
- _серая_ — вершина _проходится_ в текущей процедуре dfs;
- _черная_ — вершина _пройдена_, все итерации dfs от нее завершены.
Такие "метки" в основном используются при поиске цикла.

```cpp
enum Color {
white,
gray,
black};

vector<Color> color;
vector<vector<int>> graph

void dfs(int u){
	color[u] = gray;
	
	for (int v : graph[u]){
		if(color[v] == white){
		dfs(v)
		}
	}
	
	color[u] = black;
}

int main(){

	for (int i = 0; i < graph.size(), i++){
		if(color[i] == white){
			dfs(i, graph, color)
		}
	}
}
```

	На списке смежности: O(V + E)
	На матрице смежности: O(V^2)

# 2. Обход в ширину BFS
_Поиск в ширину_ (англ. _breadth-first search_) — один из основных алгоритмов на графах, позволяющий находить все кратчайшие пути от заданной вершины и решать многие другие задачи.
Поиск в ширину также называют _обходом_ — так же, как поиск в глубину и все другие обходы, он посещает все вершины графа по одному разу, только в другом порядке: по увеличению расстояния до начальной вершины.
## Описание алгоритма

это метод обхода графа или дерева, который начинает с начальной вершины и посещает все соседние узлы, прежде чем перейти к узлам следующего уровня. Он работает по принципу «волны», гарантируя нахождение кратчайшего пути в невзвешенных графах

## Код
```cpp
vector<int> bfs(vector<vector<int>> &graph, int start){
	vector<int> dist(graph.size(), INF);
	std::queue<int> q;
	dist[start] = 0;
	q.push(start);

	while(!q.empty()){
		int v = q.front();
		q.pop();
		for (int to : graph[v]){
			if (dist[to] == INF){
				dist[to] = dist[v] + 1;
				q.push(to);
			}
		}
	}
return dist;
}
```
## Асимптотика 
	На списке смежности: O(V + E)
	На матрице смежности: O(V^2)

# 3.  Топологическая сортировка 
Задача топологической сортировки графа звучит так: дан ориентированный граф, и требуется найти такой порядок вершин, в котором все рёбра графа вели из более ранней вершины в более позднюю.
Это может быть полезно, например, при планировании выполнения связанных задач: вам нужно одеться, в правильном порядке надев шорты (1), штаны (2), ботинки (3), подвернуть штаны (4) — как хипстеры — и завязать шнурки (5).

## Условие возможности TopSorta
	Выполняется только на ациклическом графе.
## Описание алгоритма
Заметим, что вершину, из которой не ведет ни одно ребро, можно всегда поставить последней, а такая вершина в ациклическом графе всегда есть (иначе можно было бы идти по обратным рёбрам бесконечно). Из этого сразу следует конструктивное доказательство: будем итеративно класть в массив вершину, из которой ничего не ведет, и убирать ее из графа. После этого процесса массив надо будет развернуть.
Этот алгоритм проще реализовать, обратив внимание на времена выхода вершин в `dfs`. Вершина, из которой мы выйдем первой — та, у которой нет новых исходящих ребер. Дальше мы будем выходить только из тех вершин, которые если и имеют исходящие ребра, то только в те вершины, из которых мы уже вышли.
Следовательно, достаточно просто выписать вершины в порядке выхода из `dfs`, а затем полученный список развернуть, и мы получим какую-то из корректных топологических сортировок.

```cpp
const int maxn = 1e5;

bool used[maxn];
vector<int> t;

void dfs(int v) {
    used[v] = true;
    for (int u : g[v])
        if (!used[u])
            dfs(u);
    t.push_back(v);
}

void topological_sort() {
    for (int v = 0; v < n; v++)
        if (!used[v])
            dfs(v);
    reverse(t.begin(), t.end());
}
```

## Асимптотика
	На списке смежности: O(V + E)
	На матрице смежности: O(V^2)

1. Построить вспомогательный граф, в котором направление всех рёбер изменено на противоположное.

		}

#include <bits/stdc++.h>
using namespace std;

bool dfs(int v, int p, const vector<vector<int>>& g, vector<int>& vis, vector<int>& parent, int& start, int& end) {
    vis[v] = 1;
    for (int u : g[v]) {
        if (u == p) continue;
        if (!vis[u]) {
            parent[u] = v;
            if (dfs(u, v, g, vis, parent, start, end)) return true;
        } else {
            start = u; end = v;
            return true;
        }
    }
    return false;
}

int main() {
    int n, m; cin >> n >> m;
    vector<vector<int>> g(n);
    for (int i = 0; i < m; i++) {
        int a,b; cin >> a >> b; a--; b--;
        g[a].push_back(b); g[b].push_back(a);
    }

    vector<int> vis(n,0), parent(n,-1);
    int start=-1,end=-1;
    for(int v=0;v<n;v++)
        if(!vis[v] && dfs(v,-1,g,vis,parent,start,end)) break;

    if(start==-1) { cout<<"NO cycle\n"; return 0; }

    vector<int> cycle; cycle.push_back(start);
    for(int v=end; v!=start; v=parent[v]) cycle.push_back(v);
    cycle.push_back(start);
    reverse(cycle.begin(), cycle.end());

    cout<<"YES\n";
    for(int v: cycle) cout<<v+1<<" "; cout<<"\n";
}

## 9. Алгоритм Краскала + DSU

Алгоритм поиска [минимального остовного дерева](https://neerc.ifmo.ru/wiki/index.php?title=%D0%9B%D0%B5%D0%BC%D0%BC%D0%B0_%D0%BE_%D0%B1%D0%B5%D0%B7%D0%BE%D0%BF%D0%B0%D1%81%D0%BD%D0%BE%D0%BC_%D1%80%D0%B5%D0%B1%D1%80%D0%B5#.D0.9D.D0.B5.D0.BE.D0.B1.D1.85.D0.BE.D0.B4.D0.B8.D0.BC.D1.8B.D0.B5_.D0.BE.D0.BF.D1.80.D0.B5.D0.B4.D0.B5.D0.BB.D0.B5.D0.BD.D0.B8.D1.8F "Лемма о безопасном ребре") (англ. _minimum spanning tree_, _MST_) во взвешенном [неориентированном связном графе](https://neerc.ifmo.ru/wiki/index.php?title=%D0%9E%D1%81%D0%BD%D0%BE%D0%B2%D0%BD%D1%8B%D0%B5_%D0%BE%D0%BF%D1%80%D0%B5%D0%B4%D0%B5%D0%BB%D0%B5%D0%BD%D0%B8%D1%8F_%D1%82%D0%B5%D0%BE%D1%80%D0%B8%D0%B8_%D0%B3%D1%80%D0%B0%D1%84%D0%BE%D0%B2#.D0.9D.D0.B5.D0.BE.D1.80.D0.B8.D0.B5.D0.BD.D1.82.D0.B8.D1.80.D0.BE.D0.B2.D0.B0.D0.BD.D0.BD.D1.8B.D0.B5_.D0.B3.D1.80.D0.B0.D1.84.D1.8B "Основные определения теории графов"). 
Жадный алгоритм
## Идея:
1. Сперва формируем ЛЕС из вершин.
2. Вычислим цикломатическое число (сколько ребер должно быть в оставе)
3. Отсортируем все ребра по весу(по возрастанию)
4. Далее добавляем рёбра с наименьшим весом, но так чтобы не образовывались циклы.

## DSU:

Система непересекающихся множеств (англ. _disjoint set union_) — структура данных, позволяющая объединять непересекающиеся множества и отвечать на разные запросы про них, например «находятся ли элементы aa и bbв одном множестве» и «чему равен размер данного множества».
Более формально, изначально имеется n элементов, каждый из которых находится в отдельном (своём собственном) множестве. Структура поддерживает две базовые операции:

- Объединить два каких-либо множества.
- Запросить, в каком множестве сейчас находится указанный элемент.

Обе операции выполняются в среднем _почти_ за O(1) (но не совсем).

```c++
int p[maxn], s[maxn];

int leader(int v) { // поиска лидера(корень) множества, в котором лежит вершина v
    return (p[v] == v) ? v : p[v] = leader(p[v]);
}

void unite(int a, int b) { // объединение вершин
    a = leader(a), b = leader(b);
    if (s[a] > s[b])
        swap(a, b);
    s[b] += s[a];
    p[a] = b;
}

void init(n) { // - Инициализация: каждая вершина сама по себе — отдельное множество, размер = 1.
    for (int i = 0; i < n; i++)
        p[i] = i, s[i] = 1;
}
```

## Код алгоритма Краскала

```c++
init(n); // n — число вершин

sort(edges.begin(), edges.end()); // сортируем рёбра по весу

int total_weight = 0;

for (auto [w, u, v] : edges) {

```

## 10. Алгоритм Прима

**Алгоритм Прима** — это жадный алгоритм для нахождения **минимального остовного дерева (MST)** в связном взвешенном графе.
**Основная идея:**
- Начинаем с любой вершины.
- Постепенно добавляем рёбра **минимального веса**, соединяющие уже выбранные вершины с остальными, не образуя циклы.
- Продолжаем, пока все вершины не будут включены.
- 
**Отличие от Краскала:**
- Краскал рассматривает **все рёбра** в порядке возрастания.
- Прим строит MST **по вершинам**, растущим из начальной вершины

Обычно реализуется на очереди.

## Идея 
1. Создать очередь с приоритетом, где стартовая - 0, остальные - бесконечность
2. Пока очередь не пуста.
	1. Выбрать минимальную вершину из очереди и убрать ее
	2. Просмотреть всё ребра из нее до всех вершин в очерди 
		1. Если просматриваемое ребро меньше текущего пути
			1. Запоминаем новый путь
			2. Запоминаем значение нового пути 
			3. Изменяем положение вершины в очереди 

## Псевдокод 

```
Прима(G, start):
    n = количество вершин в графе G
    used[0..n-1] = false           // вершины уже в MST
    min_edge[0..n-1] = INF         // минимальный вес ребра к MST
    parent[0..n-1] = -1            // родитель вершины в MST

        если used[u] == true:

        used[u] = true

                min_edge[v] = w_uv
                parent[v] = u
                pq.push((w_uv, v))
    // MST готова
    вернуть (parent, total_weight)
```

	На массиве: V^2

# 11. Форд-Беллман

Решает проблему дейкстры(могут быть отрицательные числа)

## Идея

- Алгоритм находит **кратчайшие пути от одной вершины** до всех остальных в графе, даже если есть рёбра с **отрицательным весом**.
- Он **последовательно “расслабляет” все рёбра** |V|-1 раз, постепенно улучшая оценки кратчайшего пути.
- Если после этого можно ещё улучшить путь — значит существует **отрицательный цикл**.    



```
Bellman-Ford(G, S):
    // Инициализация
    for each vertex v in G.V:
        dist[v] = +∞       // расстояние до вершины
        parent[v] = NIL    // предыдущая вершина на пути
    dist[S] = 0
    // Основной цикл: |V|-1 итераций
    for i = 1 to |G.V| - 1:
        for each edge (u, v) with вес w in G.E:
            Relax(u, v, w)
    // Проверка отрицательного цикла
    for each edge (u, v) with вес w in G.E:
        if dist[v] > dist[u] + w:
            return "Negative cycle exists"
    return dist, parent


// Функция Relax
Relax(u, v, w):
    if dist[v] > dist[u] + w:
        dist[v] = dist[u] + w
        parent[v] = u
```

## Асимптотика
	O(V * E)

## 12. Алгоритм DAG

- DAG — это **направленный граф без циклов**.
- Поскольку в DAG нет циклов, можно строить **топологическую сортировку**, которая задаёт порядок обхода вершин.
- Основное преимущество DAG — можно решать задачи на кратчайшие/длиннейшие пути за **O(V + E)**, а не за O(V·E) как в Беллмана-Форда.

## Идея
1. Сначала делаем **топологическую сортировку** вершин.
2. Обходим вершины в этом порядке.
3. Для каждой вершины u “расслабляем” все исходящие рёбра (u,v):
    - Если dist[v] > dist[u] + w(u,v) → обновляем dist[v].
**Почему работает:**
- В топологическом порядке, когда мы обрабатываем вершину u, все пути до u уже оптимальны.
- Поскольку DAG без циклов, путь нельзя улучшить позже, поэтому достаточно одного прохода.

## Псевдокод 


``` cpp
DAG_Shortest_Path(G, S):
    // Инициализация
    for each vertex v in G.V:
        dist[v] = +∞
        parent[v] = NIL
    dist[S] = 0

    // Топологическая сортировка
    order = TopologicalSort(G)

    // Расслабляем рёбра в топологическом порядке
    for u in order:
        for each edge (u, v) с весом w:
            if dist[v] > dist[u] + w:
                dist[v] = dist[u] + w
                parent[v] = u

    return dist, parent
```

## 13. Дейкстра
**Ограничение:** Не должно быть отрицательных ребер.
**Алгоритм жадный**

## Основная идея:
    1. Начинаем с вершины S.
    2. Постепенно выбираем вершину с минимальным текущим расстоянием до S.
    3. Расслабляем все её исходящие рёбра.
	4. Повторяем, пока не обработаем все вершины.

Асимптотика: O((E + V) logE )

## На Массиве (O(V^2))

- Используем массив dist[v] для расстояний и массив used[v] для пометки обработанных вершин.
- На каждом шаге ищем **необработанную вершину с минимальным** **dist[v]**.
- Подходит для **плотных графов** (много рёбер).
## Псевдокод 

``` c++
Dijkstra_Array(G, S):
    for each vertex v:
        dist[v] = +∞
        used[v] = false
        parent[v] = NIL
    dist[S] = 0

    for i = 1 to |V|:
        u = vertex with minimal dist[u] among not used vertices
        used[u] = true
        for each edge (u, v) с весом w:
            if dist[v] > dist[u] + w:
                dist[v] = dist[u] + w
                parent[v] = u

    return dist, parent
```

## На очереди(О(E\*logV))

- Используем **priority_queue**, хранящую пары (dist[v], v).
- Каждый раз извлекаем вершину с минимальным расстоянием.
- Для соседей выполняем Relax и добавляем их в очередь, если расстояние улучшилось.
- Подходит для **разреженных графов**.
## Псевдокод: 

``` c++
Dijkstra(G, S):
    // Инициализация
    for each vertex v in G.V:
        dist[v] = +∞
        parent[v] = NIL
    dist[S] = 0

    pq = empty priority_queue
    pq.push((0, S)) // (расстояние, вершина)

    while pq не пустая:
        (d, u) = pq.pop()
        if d > dist[u]: 
            continue  // устаревшая запись
        for each edge (u, v) с весом w:
            if dist[v] > dist[u] + w:
                dist[v] = dist[u] + w
                parent[v] = u
                pq.push((dist[v], v))

    return dist, parent
```

# 14. Флойд-Уоршелл
Поиск крайтчайших путей от всех до всех
Допускает наличие отрицательные значения и отрицательные циклы.

Алгоритм находит **кратчайшие пути между всеми парами вершин** в графе.
Позволяет работать с **отрицательными рёбрами**, но **не с отрицательными циклами** (или выявляет их).
- Основная идея:
    1. Рассматриваем вершины по порядку k = 0..n-1 как **возможный промежуточный узел**.
    2. Для каждой пары вершин (i, j) проверяем:
        - Можно ли улучшить путь i → j, пройдя через k?
        - d[i][j] = min(d[i][j], d[i][k] + d[k][j])
- После трёх вложенных циклов d[i][j] содержит **кратчайшее расстояние между всеми вершинами**.

```

FloydWarshall(G): 
    for i = 0 to n-1: // инициализация матрицы расстояний
        for j = 0 to n-1:
            if i == j: d[i][j] = 0
            else if edge i->j exists: d[i][j] = weight(i,j)
            else: d[i][j] = +∞

for i ... 0 - n - 1:
	for j ... 0 - n - 1:
		for h ... 0 - n - 1:
			d(j,h) = min(d(j,h), d(j,i) + d(i, h))
```
Асимптотика: O(V^3)

# 15. Диаметр дерева

 **Диаметр дерева** — это **длина (количество рёбер или сумма весов) самого длинного пути между двумя вершинами**.
В **невзвешенном дереве** диаметр = количество рёбер в самом длинном пути. В **взвешенном дереве** диаметр = сумма весов рёбер в самом длинном пути.

## Идея
Самый простой способ найти диаметр дерева:
1. Берём **любую вершину** u.
2. Запускаем **DFS или BFS** из u, находим вершину v, которая находится на максимальном расстоянии от u.
3. Запускаем DFS/BFS из v, находим вершину w, которая находится на максимальном расстоянии от v.
4. **Расстояние между** **v** **и** **w** — это **диаметр дерева**.
**Почему работает:**
- В дереве любое два пути не образуют цикла, и самый длинный путь обязательно начинается и заканчивается в “концах” дерева.

## Псевдокод для невзвешенного дерева на BFS

``` c++
function TreeDiameter(Tree):
    // Шаг 1: BFS/DFS из любой вершины
    u = 0
    dist1, far1 = BFS(Tree, u)   // far1 - вершина на максимальном расстоянии
    // Шаг 2: BFS/DFS из far1
    dist2, far2 = BFS(Tree, far1)
    diameter = dist2[far2]
    return diameter

function BFS(Tree, start):
    queue = empty
    queue.push(start)
    dist[start] = 0
    used[start] = true
    farthest = start

    while queue not empty:
        u = queue.pop()
        for each neighbor v of u:
            if not used[v]:
                used[v] = true
                dist[v] = dist[u] + 1
                queue.push(v)
                if dist[v] > dist[farthest]:
                    farthest = v
    return dist, farthest
```

 Для **взвешенного дерева** вместо dist[v] = dist[u] + 1 пишем dist[v] = dist[u] + w(u,v).
 
## Псевдокод для DFS
``` c++
DFS(u, parent):
    visited[u] = true       // помечаем вершину как посещённую
    for each vertex v adjacent to u:
        if not visited[v]:
            DFS(v, u)

DFS(u, parent, dist):
    visited[u] = true
    farthest_node = u
    max_dist = dist
    for each vertex v adjacent to u:
        if v != parent:           // не возвращаемся к родителю
            node, d = DFS(v, u, dist + 1)  // +1 для невзвешенного дерева
            if d > max_dist:
                max_dist = d
                farthest_node = node

    return farthest_node, max_dist
```

Сложность: **O(V)**, так как дерево содержит V-1 рёбер.

# 16. Алгоритм Куна 

## Идея:

1. - Алгоритм ищет **увеличивающие пути** (augmenting paths).
    - Увеличивающий путь — это путь, который:
    - начинается в свободной вершине Lчередует свободные и занятые рёбра заканчивается в свободной вершине R
    -  Когда найден такой путь, можно **увеличить паросочетание на 1**, “переворачивая” рёбра вдоль пути.
2. **Механизм работы**
    - Начинаем с пустого паросочетания.
    - Для каждой вершины u ∈ L пытаемся найти увеличивающий путь через **DFS**.
    - Если путь найден, обновляем паросочетание, рекурсивно изменяя пары вдоль пути.
    - Если путь не найден (DFS “заканчивается в левой доле”), текущая вершина u не может быть добавлена в паросочетание.
3. **Рекурсивное переворачивание рёбер**
    - Свободные рёбра идут L → R.
    - Занятые рёбра можно воспринимать как R → L во время DFS.
    - После успешного DFS все рёбра на увеличивающем пути логически переворачиваются, фиксируя новые пары.    
4. **Повторяем для всех вершин L**
    - Каждый успешный DFS увеличивает размер паросочетания на 1.
    - После прохода всех вершин получаем **максимальное паросочетание**.
	``` c++
Input: G = (L ∪ R, E)  // двудольный граф
Output: matchR[v] для всех v ∈ R и размер максимального паросочетания

// Инициализация
matchR[v] = NIL for all v in R
maxMatching = 0

// DFS для поиска увеличивающего пути
function tryKuhn(u):      // u ∈ L
    if visited[u]:
        return false
    visited[u] = true

    for each v in adj[u]:  // идём по рёбрам L → R
        if matchR[v] == NIL or tryKuhn(matchR[v]):  // рекурсивно освобождаем R
            matchR[v] = u       // обновляем паросочетание
            return true         // путь найден

    return false                // путь не найден (DFS “закончился в L”)

// Основной цикл по всем вершинам L
for u in L:
    visited[u] = false
    if tryKuhn(u):
        maxMatching += 1

return maxMatching, matchR
	```

## Асимптотика 
	O(Vl * E) 
## Форд-Фалкерсон (Поток)

прямое ребро - сколько капасити осталось, обратное сколько пустили. Если дфс прошел через обратное ребро, делаем то же самое, что и для прямого

1. **Инициализируем поток**: f(u,v) = 0 для всех рёбер (u,v).
2. **Пока существует увеличивающий путь** P из истока s в сток t в остаточном графе:

##  Алгоритм Эдмондса-Карпа
- Это **модификация Форда-Фалкерсона**, которая всегда выбирает **кратчайший путь по количеству рёбер** из s в t (BFS вместо DFS).
- **Преимущество:** гарантированная **полиномиальная сложность O(V * E²)**, в то время как обычный DFS в Форда-Фалкерсона может работать экспоненциально на графах с дробными пропускными способностями.
``` cpp
function EdmondsKarp(G, s, t):
    f(u,v) = 0 for all (u,v) in E
    while exists shortest path P from s to t in residual graph G_f (BFS):
        c_min = min(c_f(u,v) for all (u,v) in P)
        for each edge (u,v) in P:
            f(u,v) += c_min
            f(v,u) -= c_min
    return sum(f(s,v) for v in adj[s])
```

# 18. Форд-Фалкерсон для парсоч

1. У нас есть **двудольный граф** G = (L ∪ R, E).
2. Для поиска максимального паросочетания мы **строим сеть потока**:
    - Добавляем **исток** **s** и соединяем его с каждой вершиной левой доли L ребром capacity = 1.
    - Добавляем **сток** **t** и соединяем каждую вершину правой доли R ребром capacity = 1.
    - Все рёбра исходного графа L → R получают capacity = 1.
3. **Запускаем алгоритм Форда-Фалкерсона** для нахождения максимального потока из s в t.
4. **Максимальный поток = размер максимального паросочетания.**
 Каждый поток 1 по ребру L → R соответствует одному ребру паросочетания.

## Алгоритм

1. **Строим сеть потока**:
	s → L (capacity = 1)
	L → R (capacity = 1)
	R → t (capacity = 1)
2. **Инициализируем остаточные capacities:**
    - Cf(u,v) = capacity(u,v)
    - Cf(v,u) = 0 (для обратного ребра)
3. **Пока существует увеличивающий путь из s в t**:
    - Находим путь P (DFS или BFS) с Cf(u,v) > 0
    - Находим минимальную пропускную способность на пути: c_min = min(Cf(u,v))
    - Для каждого ребра (u,v) на пути:
        - Cf(u,v) -= c_min  (прямое ребро)
        - Cf(v,u) += c_min  (обратное ребро)
4. **Повторяем**, пока не останется пути.

##  Построение паросочетания
- После завершения алгоритма Форда-Фалкерсона:
    - Проверяем рёбра между L и R:
        - Если поток прошёл через ребро (т.е. Cf(u,v) == 0 для исходного ребра L → R), то добавляем это ребро в паросочетание.

## Псевдокод
``` c++
function MaxBipartiteMatching(G, L, R):
    // 1. Строим сеть потока
    add vertex s, t
    for u in L:
        add edge (s,u) with capacity = 1
    for v in R:
        add edge (v,t) with capacity = 1
    for (u,v) in E:  // исходные рёбра
        add edge (u,v) with capacity = 1

    // 2. Инициализация остаточной пропускной способности
    for each edge (u,v):
        Cf(u,v) = capacity(u,v)
        Cf(v,u) = 0

    max_flow = 0
    // 3. Алгоритм Форда-Фалкерсона
    while exists path P from s to t in residual graph:
        c_min = min(Cf(u,v) for (u,v) in P)
        for (u,v) in P:
            Cf(u,v) -= c_min
            Cf(v,u) += c_min
        max_flow += c_min

    // 4. Построение паросочетания
    matching = []
    for u in L:
        for v in adj[u]:
            if original edge (u,v) and Cf(u,v) == 0:  // поток прошёл
                matching.append((u,v))

    return max_flow, matching
```

## Идея алгоритма (DFS + Low-Link)

2. Для каждой вершины v храним:
    - tin[v] — время входа DFS в вершину (номер посещения).
    - low[v] — минимальное время входа, которое можно достичь из поддерева DFS вершины v (включая обратные рёбра).
3. **Правила:**
    - **Мост:** ребро (v,u) — мост, если low[u] > tin[v].
        - Значит, из поддерева u нельзя вернуться в предков v.
    - **Точка сочленения:**
        - v — корень DFS с ≥2 детьми → точка сочленения.
        - v ≠ корень: если существует ребёнок u с low[u] ≥ tin[v], то v — точка сочленения.

```cpp
timer = 0

function DFS(v, parent):
    visited[v] = true
    tin[v] = low[v] = timer
    timer += 1
    children = 0

    for to in adj[v]:
        if to == parent:
            continue
        if visited[to]:
            // обратное ребро
            low[v] = min(low[v], tin[to])
        else:
            DFS(to, v)
            low[v] = min(low[v], low[to])
            if low[to] > tin[v]:
                print("Мост:", v, to)
            if parent != NIL and low[to] >= tin[v]:
                print("Точка сочленения:", v)
            children += 1

    if parent == NIL and children > 1:
        print("Точка сочленения (корень):", v)

// Запуск DFS
for v in V:
    if not visited[v]:
        DFS(v, NIL)
```

# 20.  Жадный алгоритм для поиска раскраски графа.

## 1. Идея
- **Задача:** раскрасить вершины графа так, чтобы **никакие две соседние вершины не имели одинаковый цвет**.
- **Цель жадного алгоритма:** использовать как можно **меньшее количество цветов** (не обязательно оптимальное, но часто близко к минимальному).
- Это **жадный алгоритм**, потому что для каждой вершины выбирается **первый доступный цвет**, не задумываясь о будущем.

## 2. Шаги алгоритма
1. Задаём граф G = (V, E) и массив цветов color[v] = -1 (не раскрашено).
2. Раскрашиваем первую вершину в цвет 0: color[v0] = 0.
3. Для каждой следующей вершины u:
    - Определяем **цвета соседей**.
    - Назначаем **первый цвет**, который не используется соседями.
4. Повторяем для всех вершин.
> Жадный алгоритм **не гарантирует минимальное число цветов**, но работает быстро: O(V + E).

## Псевдокод
``` cpp
function GreedyColoring(G):
    n = number of vertices in G
    color = [-1] * n          // все вершины не раскрашены
    color[0] = 0              // первая вершина в цвет 0

    for u = 1 to n-1:         // остальные вершины
        available = [true] * n  // массив доступных цветов

        // отмечаем цвета соседей как недоступные
        for v in adj[u]:
            if color[v] != -1:
                available[color[v]] = false

        // выбираем первый доступный цвет
        for c = 0 to n-1:
            if available[c]:
                color[u] = c
                break

    return color               // раскраска вершин
```
